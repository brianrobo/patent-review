이번 단락은 장치가 비정상적인 명령을 감지했을 때 취하는 구체적인 대응 메커니즘을 다루고 있습니다. 특히 보안 강도를 낮추려는 시도(Ciphering 해제 등)에 대한 장치의 방어 동작이 핵심입니다.
VS Code에서 바로 확인하실 수 있도록 end 문법을 준수하여 작성했습니다.
1. 보안 위협 탐지 및 대응 로직 (Flowchart)
장치가 명령의 부적절함을 인지하고 단계별 대응(무시, 사용자 알림, 차단)을 실행하는 과정입니다.

2. 장치의 단계별 대응 전략 (Mindmap)
Summary에 언급된 다양한 보안 조치들을 강도에 따라 분류한 다이어그램입니다.

3. 핵심 내용 요약
이 단락에서 Apple이 강조하는 **'지능형 보안 조치'**는 다음과 같습니다.
 * 일관성 검사 (Consistency Check): 현재 접속한 MNO(통신사)와 지역적 특성에 맞지 않는 명령(예: 선진국 도심에서 암호화 해제 요청)이 오면 즉시 위협으로 간주합니다.
 * 강제 거부 (Hard Rejection):
   * 암호화 미적용(No Ciphering) 또는 약한 암호화 요청 시 무시.
   * 보안이 취약한 하위 프로토콜로의 강제 리다이렉트 거부.
   * 암호화되지 않은 식별 정보(IMSI 등) 요청 거부.
 * 블랙리스트 관리 (Blacklisting): 단순히 연결을 끊는 것에 그치지 않고, 해당 기지국을 블랙리스트에 추가하여 영구적으로 차단하거나 일정 기간 통신을 금지한 뒤 새로운 정상 기지국을 검색(Scanning)합니다.
 * 사용자 개입 (User Involvement): 중대한 보안 변경이 있을 경우 사용자에게 경고를 보내고, 계속 진행할지 여부를 직접 선택하게 합니다.
